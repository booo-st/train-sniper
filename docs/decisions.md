# 주요 결정 사항

---

## 2026-06-22 — 좌석 옵션을 드롭다운에서 체크박스로 변경

기존 4가지 옵션(special-first / general-first / general-only / special-only) 드롭다운 → 일반실 / 특실 체크박스 2개로 단순화.  
둘 다 체크 = `general-first`, 하나만 = `*-only`. `special-first` 옵션 제거 (양쪽 다 선택 시 항상 일반실 우선).  
선택지가 많은 드롭다운보다 의도가 직관적으로 전달됨.

---

## 2026-06-22 — 잡 영속성 추가 (서버 재시작 후 복원)

기존에는 서버 종료 시 실행 중인 모든 잡이 사라졌음.  
`settings.local.json`의 `"jobs"` 키에 잡 파라미터와 완료 상태를 저장, 서버 재시작 시 복원.  
in-memory 전용 필드(thread, stop_event)는 제외하고 재시작 시 새로 할당.

---

## 2026-06-22 — 최소 조회 간격 10초 → 1초

사용자가 빠른 감시 주기를 원할 때 선택 가능하도록 완화.  
너무 짧으면 코레일/SRT 서버에 부하 우려가 있으나, 사용자 책임으로 판단.

---

## 2026-06-22 — 과거 날짜 잡 생성/재개 차단

날짜가 지난 잡을 실행해도 열차가 없어 의미 없는 루프만 돎.  
생성 시와 워커 시작 시 두 곳에서 오늘 이전 날짜 체크, 에러 반환 또는 즉시 중지.

---

## 2026-06-16 — 웹 UI 비밀번호 게이트 추가

ngrok으로 외부 노출 시 가족 외 접근을 막기 위해 단일 비밀번호 인증 추가.  
사용자별 계정 시스템 없이 대문 비밀번호 방식 채택.  
`ui_password` 미설정 시 로컬 사용 그대로 유지 (하위 호환).

---

## 2026-06-16 — 텔레그램 알림에 계정 이름 포함

여러 프로필로 동시 스나이핑 시 어떤 계정으로 예약됐는지 알림만으로 구분 가능하게.  
계정 ID로 `config.json` 프로필을 역조회해 이름 표시. ID 직접 노출 방지.

---

## 2026-06-16 — 계정 프로필 시스템 추가

매번 계정 정보를 입력하는 불편 해소.  
`config.json`에 `ktx_profiles` / `srt_profiles` 배열로 여러 계정 사전 정의,  
웹 UI 드롭다운에서 선택 후 "적용" 버튼으로 활성 계정 전환.

---

## 2026-06-16 — Vercel 배포 불채택

Vercel은 서버리스 함수 기반이라 스나이핑 워커(장기 실행 백그라운드 스레드)를 지원하지 않음.  
대신 ngrok + 로컬 실행 방식 유지.

---

## 초기 — 표준 라이브러리만으로 HTTP 서버 구현

Flask 등 웹 프레임워크 없이 `http.server` + `threading` 표준 라이브러리만 사용.  
의존성을 최소화해 설치 간소화.
