# 주요 결정 사항

---

## 2026-06-16 — 웹 UI 비밀번호 게이트 추가

ngrok으로 외부 노출 시 가족 외 접근을 막기 위해 단일 비밀번호 인증 추가.  
사용자별 계정 시스템 없이 대문 비밀번호 방식 채택.  
`ui_password` 미설정 시 로컬 사용 그대로 유지 (하위 호환).

---

## 2026-06-16 — 텔레그램 알림에 계정 이름 포함

여러 프로필로 동시 스나이핑 시 어떤 계정으로 예약됐는지 알림만으로 구분 가능하게.  
계정 ID로 `config.json` 프로필을 역조회해 이름 표시. ID 직접 노출 방지.

---

## 2026-06-16 — 계정 프로필 시스템 추가

매번 계정 정보를 입력하는 불편 해소.  
`config.json`에 `ktx_profiles` / `srt_profiles` 배열로 여러 계정 사전 정의,  
웹 UI 드롭다운에서 선택 후 "적용" 버튼으로 활성 계정 전환.

---

## 2026-06-16 — Vercel 배포 불채택

Vercel은 서버리스 함수 기반이라 스나이핑 워커(장기 실행 백그라운드 스레드)를 지원하지 않음.  
대신 ngrok + 로컬 실행 방식 유지.

---

## 초기 — 표준 라이브러리만으로 HTTP 서버 구현

Flask 등 웹 프레임워크 없이 `http.server` + `threading` 표준 라이브러리만 사용.  
의존성을 최소화해 설치 간소화.
